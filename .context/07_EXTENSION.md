# 07 — VS Code Extension

## Overview

```
Extension ID: gitphone-vscode
Language:     TypeScript (strict)
Entry point:  src/extension.ts

Extension runs on developer's local machine.
Purpose: Watch file saves → compute diffs → sync to backend.
```

---

## Architecture

```
File Save (onDidSaveTextDocument)
    │
    ▼
diffEngine.ts
  ├── CRLF → LF normalization
  ├── Binary file detection (isBinary)
  ├── Minified file detection (.min.js etc)
  ├── Size check (>10MB = reject)
  ├── Read local cache (last committed version)
  ├── Compute diff (diff npm)
  └── Return { diff, base_sha, is_binary, change_type }
    │
    ▼
fileWatcher.ts
  ├── Gets git repo from .git/config (auto-detect)
  ├── Populates repo, branch, change_type
  └── Sends to api.ts
    │
    ▼
api.ts
  ├── POST /sync-file (with X-Api-Key + X-Telegram-Id)
  └── Body: { filepath, diff, full_content?, base_sha, repo, branch, is_binary, file_size, change_type }
    │
    ▼
Backend stores in staged_files table
```

## Extension Components

### src/extension.ts (Entry Point)
```
activate():
  1. Check schema version (GET /version)
  2. Load config from globalState
  3. Register all commands
  4. Init fileWatcher, statusBar, stagedFilesProvider
  5. Start health check polling (every 5 min)
  6. Show status bar item

Commands registered:
  gitphone.connect          - Open setup panel
  gitphone.disconnect       - Clear config + staged dialogs
  gitphone.refresh          - Force re-sync all files
  gitphone.openBot          - Open Telegram bot link
  gitphone.commitFromBar    - Trigger commit flow
  gitphone.syncNow          - Manual sync of current file
  gitphone.openCommitUI     - Open fallback web commit UI
```

### src/setupPanel.ts
```
Webview panel with fields:
  1. API Key (hidden, generated by backend)
  2. Telegram ID (numeric, from @userinfobot)
  3. GitHub Token (PAT or leave blank for Device Flow)
  4. Working Directory (optional)
  5. Connect/Register button

On connect:
  POST /register → gets api_key back
  Stores in ExtensionContext.globalState
  Sets status bar to connected

Also supports OAuth Device Flow:
  Button: "Sign in with GitHub (Device Flow)"
  → Opens browser to github.com/login/device
  → Bot polls and receives token
  → Extension detects auth via /register
```

### src/fileWatcher.ts
```
Listens to:
  vscode.workspace.onDidSaveTextDocument
  vscode.workspace.onDidCreateFiles
  vscode.workspace.onDidDeleteFiles
  vscode.workspace.onDidRenameFiles

For each event:
  1. Compute filepath relative to workspace root
  2. Skip .git/**, node_modules/**, .vscode/**
  3. Auto-detect git repo from .git/config
  4. Call diffEngine.ts to compute diff
  5. Post to backend via api.ts
```

### src/diffEngine.ts
```
computeDiff(filePath, oldContent?, newContent):
  1. Normalize CRLF to LF
  2. Detect binary (null bytes, Unicode replacement)
  3. Detect minified (single long line, .min.js pattern)
  4. Check size (>10MB = reject)
  5. Read localCache for old version
  6. Use diff.createTwoFilesPatch() for diff
  7. Return { diff?, full_content?, base_sha, is_binary, change_type }

Change type detection:
  - File exists locally but not in cache → 'create'
  - File exists in cache, local deleted → 'delete'
  - Both exist → 'modify'
```

### src/api.ts
```
HTTP client using axios with:
  baseURL: backend URL (from config)
  headers: { X-Api-Key, X-Telegram-Id, Content-Type: application/json }

Methods:
  register(telegramId, githubToken, workingDir?) → { api_key }
  syncFile(filepath, diff, base_sha, ...) → { id, status }
  getStagedFiles(status?) → { files }
  getVersion() → { version, schema_version }
  health() → { status }
```

### src/stagedFilesProvider.ts (Sidebar TreeView)
```
Provides sidebar view showing staged files using vscode.git API:
  - File tree with status indicators
  - Manual sync button per file
  - Refresh button
  Uses vscode.extensions.getExtension('vscode.git').exports
```

### src/statusBar.ts
```
Always visible bottom bar states:

  🟢 GitPhone — Connected
  Shows when successfully registered to backend

  🟢 GitPhone — 3 file(s) staged
  Shows when files are staged

  ⚠️ GitPhone — Connected
  Warning state (errors, version mismatch)

  🔴 GitPhone — Disconnected
  Not registered, no config saved
```

### src/localCache.ts
```
Stores last-committed version of each file locally:
  filepath → { content, sha, timestamp }
  Stored in ExtensionContext.globalState
  Used by diffEngine.ts to compute diff against
  Updated after successful commit

Purpose: Avoid GitHub API calls on every file save.
GitHub API called ONLY for:
  → SHA fetch at first sync
  → SHA fetch post-commit
  → Never on file save ✅
```

### src/encryption.ts
```
AES-256 encryption for API key storage:
  Encrypts API key with machine-specific key
  decrypt(text: string) → plaintext key
  encrypt(text: string) → ciphertext
  FUTURE: Implementation pending
  Currently stores in VS Code's secure storage
```

## 10MB Hard Limit

```
// diffEngine.ts
const MAX_FILE_SIZE = 10 * 1024 * 1024; // 10MB

if (fileSize > MAX_FILE_SIZE) {
  vscode.window.showWarningMessage(
    `⚠️ ${filepath} skipped — ` +
    `exceeds 10MB limit. ` +
    `Add to .gitignore instead.`
  );
  return null; // Skipped
}
```
